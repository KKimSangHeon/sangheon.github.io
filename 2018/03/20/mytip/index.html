html 파일 : /home1/irteam/apps/apache/htdocs

서버의 아파치 실행

sudo /sbin/service httpd start
sudo /sbin/apachectl start
sudo /sbin/service httpd status
sudo /sbin/apachectl status
참고 : http://library.gabia.com/contents/infrahosting/3513

아파치 모듈 추가

http://prettymonkey02.tistory.com/1051

성공한 방법

  1. /home1/irteam/apps/apache/bin 에서 “./apxs -n helloworld -g” 명령어 입력하면 helloworld 디렉터리가 생성된다.

  2. 생성된 helloworld 디렉터리 들어간 후 vi mod_helloworld.c 를 참고하여 명령들을 실행.(명령들은 다음과같다)
    /home1/irteam/apps/apache/bin 디렉토리로 이동 후 “#sudo ./apxs -c -i helloworld/mod_helloworld.c” 실행
    /home1/irteam/apps/apache/conf 디렉토리로 이동 후 “vi httpd.conf” 열기
    적당한 곳에 다음 2-4번의 내용 추가 ( 약 156라인)
    LoadModule helloworld_module modules/mod_helloworld.so


    SetHandler helloworld

  3. /home/irteam/apps/apache/bin 디렉토리에 들어간 후 “# sudo ./apachectl restart” 입력

  4. 테스트를 위해 “lynx -mime_header http://localhost/helloworld“ 입력

대문자로 모듈을 생성할 경우 안되는것 같다…

C파일을 수정하였을 경우
C 파일을 수정 한 후에는 위의 2-1번만 실행 후 3번으로 거치면 된다.

시도해봤지만 안되는 방법
apxs -n hello -g 를 하면 so 파일이 /usr/lib64/httpd/modules/mod_hello.so 에 생성되는데 “sudo mv /usr/lib64/httpd/modules/mod_hello.so 내 modules 디렉토리” 명령어를 통해 so 파일을 옮겨준다.
“sudo chown irteam:irteam mod_hello.so” 명령어를 통해 소유자와 관리자를 다른것과 동일하게 변경한다.

LoadModule hello_module /modules/mod_hello.so 이라는 문자열이 /etc/httpd/conf/httpd.conf
에 생성되는데 내 경로의 conf/httpd.conf 149쯔음 사이에 끼워 넣어준다.
